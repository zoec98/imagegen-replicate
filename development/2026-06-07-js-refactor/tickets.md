# JavaScript Refactor Tickets

Source proposal: `development/refactors/2026-06-07-js-refactor.md`.

## Decisions Applied

- Use Vite with vanilla JavaScript.
- Keep browser runtime dependencies at zero initially.
- Store JavaScript source modules under `src/imagegen/frontend/`.
- Continue serving the generated browser target as
  `src/imagegen/static/app.js`.
- Keep generated `app.js` committed so the Flask app does not require Node at
  runtime.
- Use Vitest with jsdom for DOM-focused JavaScript tests.
- Use ESLint for correctness linting and Prettier for formatting.
- Do not introduce React, Vue, Svelte, TypeScript, Babel, jQuery, or runtime
  state-management libraries in the initial migration.

## Ticket 1: Add JavaScript Toolchain

### Goal

Introduce a small dev-only Node toolchain for building, testing, linting, and
formatting browser JavaScript without changing runtime behavior.

### Scope

- Add a root `package.json`.
- Add a lockfile generated by the selected package manager.
- Add Vite, Vitest, jsdom, ESLint, `@eslint/js`, and Prettier as dev
  dependencies.
- Add scripts for `js:build`, `js:test`, `js:lint`, `js:format`, and
  `js:check`.
- Add `vite.config.js`.
- Add `eslint.config.js`.
- Add Prettier configuration.
- Configure Vite to output `src/imagegen/static/app.js`.
- Configure Vite with `emptyOutDir: false`, `minify: false`, and sourcemaps.

### Acceptance Criteria

- `npm run js:build` creates or updates `src/imagegen/static/app.js`.
- `npm run js:test` runs Vitest.
- `npm run js:lint` runs ESLint against frontend source and JS tests.
- `npm run js:format` formats frontend source and JS tests.
- `npm run js:check` runs lint, tests, and build.
- The Flask template can keep loading `/static/app.js`.
- Running the Flask app does not require Node.

### Verification

- `npm run js:check`
- `uv run pytest`
- `uv run ruff check src tests`
